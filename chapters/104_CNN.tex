\section{Computer Vision with CNNs}
\label{sec:cnn}

\subsection{How a CNN Reads an Image}

A \textbf{Convolutional Neural Network} (CNN) is a type of neural network that is designed to work with images. The basic idea is quite clever: instead of looking at every single pixel separately, the CNN slides small filters across the image to detect patterns like edges, textures, and shapes. These filters start simple (detecting lines and corners in the first layers) and become more complex deeper in the network (detecting whole objects or structures) \cite{lecun1998cnn}.

The name ``convolutional'' comes from the mathematical operation used to apply these filters. What makes CNNs so efficient is that the same filter is reused across the entire image. This is called \textbf{weight sharing}, and it means the network does not need a separate set of parameters for every pixel. That is why CNNs can handle large images without needing billions of parameters.

A typical CNN has a few types of layers that work together:

\begin{itemize}
\item \textbf{Convolution layers:} these apply the filters and extract features from the image.
\item \textbf{Activation layers (ReLU):} these add non-linearity, which lets the network learn complex patterns instead of just straight lines.
\item \textbf{Pooling layers:} these shrink the feature maps to make the network faster and more robust to small shifts in the image.
\item \textbf{Fully connected layers:} at the end, these combine all the extracted features to produce a final output, like a classification or a set of visual features for another system to use.
\end{itemize}

CNNs have been behind many of the biggest breakthroughs in computer vision. \textbf{AlexNet} won the ImageNet competition in 2012 and started the deep learning revolution in vision \cite{krizhevsky2012alexnet}. A few years later, \textbf{ResNet} showed that networks with over 100 layers could be trained successfully by using skip connections \cite{he2016resnet}. In medical imaging, \textbf{U-Net} became the go-to architecture for tasks like segmenting organs and lesions in scans \cite{ronneberger2015unet}.

\subsection{Why CNNs Are Good for Endoscopic Video}

Endoscopic images are not easy for computers to work with. The lighting changes all the time, there are bright reflections on wet surfaces, the camera moves quickly and sometimes blurs the image, and the colon wall can look very different from one patient to another. On top of that, polyps can be quite small and hard to tell apart from normal folds in the tissue.

Despite all these challenges, CNNs have shown impressive results in endoscopy. A large meta-analysis by Nazarian et al. found that deep learning models can detect colorectal polyps with a \textbf{sensitivity above 90\,\%} \cite{nazarian2021jmir}. Multiple clinical trials have confirmed that adding a CNN-based detection system to the colonoscopy workflow actually helps doctors find more polyps \cite{wang2019cade,repici2020cade}.

What makes CNNs well suited for this task is their ability to learn visual features that are robust to all the noise and variation mentioned above. With enough training data and good augmentation techniques (like randomly changing brightness, adding blur, or flipping images), a CNN can learn to focus on the important patterns and ignore distractions like reflections or motion blur.

In our project, the CNN is not used as a standalone detector. Instead, it acts as a \textbf{visual encoder} that turns the raw camera image into a compact set of features. These features are then passed to the Deep RL agent so it can understand the scene and decide what to do next.

\subsection{What Our Agent Sees: Colon Walls and Polyp Targets}

The agent in our simulation receives two types of visual input. The first is the \textbf{main camera view}, which shows the colon wall, the lumen (the open space inside the colon), and any polyps that might be visible. This is similar to what a real endoscopist sees on the screen during a procedure. The second input can include additional information like depth or tool position, depending on the task phase.

For the navigation phase, the CNN needs to help the agent understand the shape of the tunnel ahead. Is the colon bending left or right? Is there an obstacle? Is the view clear or blocked by fluid? For the extraction phase, the CNN needs to help the agent see where the polyp is relative to the snare tool, how far away it is, and whether the snare is in the right position to close.

The important point is that the CNN does not need to be perfect. It does not need to classify the polyp type or measure its exact size. It just needs to produce features that are \textbf{good enough} for the Deep RL agent to make reasonable decisions. This is a more forgiving task than pure detection, and it means we can use a relatively simple CNN architecture without sacrificing too much performance.
