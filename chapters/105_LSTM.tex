\section{Temporal Memory with LSTMs}
\label{sec:lstm}

\subsection{Why One Frame is Not Enough}

When a doctor performs a colonoscopy, they do not make decisions based on a single frozen image. They watch the video, notice how the scene changes over time, and use that context to understand what is happening. For example, if the camera just passed a fold, the doctor knows that the view will open up in a moment. If the polyp was visible two seconds ago but disappeared, the doctor knows it is probably just behind the current camera angle.

An Deep RL agent that only looks at the current frame misses all of this context. It would be like trying to watch a football match by looking at one random photo every few seconds. You would not understand who is attacking, who is defending, or what just happened. That is why we need a way to give the agent a \textbf{short-term memory} so it can connect what it sees now with what it saw a few steps ago.

This is where \textbf{Long Short-Term Memory} (LSTM) networks come in. They are a type of recurrent neural network designed to remember important information across several time steps and forget the rest \cite{hochreiter1997lstm}.

\subsection{How LSTMs Remember What Just Happened}

A normal neural network processes each input independently. It has no memory of what came before. A \textbf{recurrent neural network} (RNN) tries to fix this by passing information from one time step to the next. But standard RNNs have a big problem: they struggle to remember things from more than a few steps back because the learning signal fades away during training. This is called the \textbf{vanishing gradient problem}.

LSTMs were invented specifically to solve this issue. They use a clever system of three \textbf{gates} that control the flow of information:

\begin{itemize}
\item \textbf{Forget gate:} decides which parts of the old memory to throw away. If something is no longer relevant (like a fold the camera already passed), the forget gate removes it.
\item \textbf{Input gate:} decides which new information to store. If the agent just spotted the polyp, the input gate makes sure that information is saved.
\item \textbf{Output gate:} decides what part of the memory to use right now for making a decision.
\end{itemize}

The combination of these gates gives the LSTM the ability to keep useful information for many time steps and discard the rest. This makes it much better than a standard RNN for tasks where the context from a few seconds ago still matters.

\subsection{Helping the Agent Keep Track During a Procedure}

In our project, the LSTM sits between the CNN and the Deep RL policy. The CNN produces visual features from the current camera frame, and the LSTM receives those features along with its own memory from the previous steps. It then produces an updated hidden state that captures both what the agent sees now and what it remembers from the recent past. The Deep RL policy uses this hidden state to choose the next action.

This setup helps the agent in several concrete ways:

\begin{itemize}
\item During \textbf{navigation}, the LSTM helps the agent understand which direction the colon is curving, even when the current view is unclear. If the last few frames showed the lumen moving to the left, the agent can anticipate the turn.
\item During \textbf{extraction}, the LSTM helps the agent track the polyp position over time, even if it briefly goes out of view. It also helps the agent remember whether the snare has already been extended or not.
\item In both phases, the LSTM \textbf{smooths the agent's behaviour} over time, reducing sudden and erratic movements that could happen if the agent only reacted to individual frames.
\end{itemize}
