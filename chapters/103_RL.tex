\subsection*{Introduction}

\cite{mnih2015dqn,silver2016alphago}.

This chapter introduces the fundamental concepts of reinforcement learning, the mathematical framework of Markov Decision Processes, core solution methods, and key applications. The discussion emphasizes concepts necessary for understanding modern policy-based methods such as \textbf{Proximal Policy Optimization (PPO)}, which is covered in detail in the following chapter.

In procedural domains like endoscopy and surgical robotics, RL is attractive because it
explicitly models \textbf{sequences of actions} under uncertainty. At the same time,
these domains are safety-critical, so training usually has to stay in simulation or rely
on carefully controlled data instead of free exploration in the real world.

\subsection{Core Concepts and Terminology}
A reinforcement learning system comprises an \textbf{agent}, an \textbf{environment}, and their interaction. The agent observes the current state of the environment, selects and executes an action, and receives both a \textbf{reward signal} and the next state. The reward encodes the immediate desirability of the action, while the state transition allows the agent to learn the consequences of its choices.

Key concepts include:

\begin{itemize}

\item \textbf{State ($s$):} A representation of the current configuration of the environment. States can be discrete or continuous, and may be fully or partially observable.

\item \textbf{Action ($a$):} A choice the agent can make. Actions are typically drawn from a finite action space, though continuous action spaces are also common.

\item \textbf{Reward ($r$):} A scalar signal indicating the immediate quality of the action taken in a given state. Rewards guide learning toward desired behaviors.

\item \textbf{Policy ($\pi$):} A mapping from states to actions, either deterministic ($a = \pi(s)$) or stochastic ($a \sim \pi(\cdot|s)$). Learning a good policy is the primary objective of RL.

\item \textbf{Value function ($V(s)$):} The expected cumulative future reward starting from state $s$ under a given policy. It estimates the long-term benefit of being in that state.

\item \textbf{Action-value function ($Q(s,a)$):} The expected cumulative future reward for taking action $a$ in state $s$ and then following the policy thereafter.

\item \textbf{Return ($G_t$):} The cumulative discounted reward from time $t$ onward: $G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \ldots$, where $\gamma \in [0,1]$ is the discount factor.

\end{itemize}

\subsection{Markov Decision Process (MDP)}
A Markov Decision Process provides the mathematical foundation for most reinforcement learning algorithms. An MDP is defined by a tuple $(S, A, P, R, \gamma)$, where:

\begin{itemize}

\item $S$ is the set of states.

\item $A$ is the set of actions.

\item $P(s'|s,a)$ is the state transition probability, giving the probability of moving to state $s'$ after taking action $a$ in state $s$.

\item $R(s,a,s')$ is the reward function, specifying the immediate reward for the transition.

\item $\gamma$ is the discount factor, weighting immediate rewards more heavily than distant future rewards.

\end{itemize}

The Markov property states that future transitions depend only on the current state and action, not on the history of prior states. This assumption simplifies analysis and enables efficient learning algorithms, though in practice many real environments are only approximately Markovian.

\subsection{Value-Based Methods}
Value-based methods learn an estimate of the value function and derive the policy implicitly by selecting actions that lead to higher-value states. The value function satisfies the Bellman equation:

$$V^\pi(s) = \mathbb{E}\left[r + \gamma V^\pi(s')\right]$$

Key value-based algorithms include:

\begin{itemize}

\item \textbf{Q-Learning:} An off-policy algorithm that learns the optimal action-value function $Q^*(s,a)$ by iteratively applying the Bellman optimality operator. Q-learning does not require knowledge of the transition model and can learn from experience \cite{sutton2018reinforcement}.

\item \textbf{SARSA (State-Action-Reward-State-Action):} An on-policy temporal difference method that updates the value function based on the action actually taken by the current policy, rather than the optimal action.

\item \textbf{Temporal Difference (TD) Learning:} A family of methods that update value estimates using the difference between the current estimate and a bootstrapped target that includes a reward and the estimate at the next state \cite{sutton2018reinforcement}.

\end{itemize}

Value-based methods work well when the action space is discrete and the state space is
manageable, but they can struggle with continuous actions or very high-dimensional
inputs.

\subsection{Policy-Based Methods}
Policy-based methods directly parametrize and optimize the policy $\pi_\theta(a|s)$. 
Unlike value-based methods, they do not need to learn a value function first, which makes
them more flexible for continuous action spaces and high-dimensional problems.

The policy gradient theorem provides the foundation:

$$\nabla_\theta J(\theta) = \mathbb{E}\left[\nabla_\theta \log \pi_\theta(a|s) Q^\pi(s,a)\right]$$

This result shows that the gradient of the expected return with respect to the policy parameters can be estimated using samples from the environment.

Key policy-based approaches include:

\begin{itemize}

\item \textbf{REINFORCE:} A simple Monte Carlo policy gradient method that uses the full return as the estimate of the action value \cite{williams1992reinforce}.

\item \textbf{Actor-Critic Methods:} Combine a policy-based actor with a value-based critic. The critic learns a value function to reduce the variance of policy gradient estimates, improving sample efficiency.

\item \textbf{Trust Region Methods:} Constrain policy updates to remain within a trust region around the current policy, ensuring stable learning \cite{schulman2015trpo,schulman2017ppo}.
\end{itemize}

\subsection{Reward Design and Safety Constraints}
In safety-critical applications, the reward function must be designed carefully.
Poorly specified rewards can lead to unintended behaviors that maximize the numerical
objective while violating practical constraints. Common approaches to mitigate this
include:

\begin{itemize}
\item \textbf{Reward shaping} to provide dense feedback while preserving the final goal.
\item \textbf{Constraints and penalties} for unsafe states or actions.
\item \textbf{Human-in-the-loop evaluation} to validate behavior beyond scalar scores.
\item \textbf{Training in simulation} before any real-world deployment.
\end{itemize}

\subsection{Exploration and Exploitation}
A fundamental challenge in reinforcement learning is balancing exploration (trying new actions to discover their effects) and exploitation (performing actions known to yield high rewards). Pure exploitation can lead to suboptimal policies that become stuck in local maxima, while excessive exploration wastes samples on low-reward actions.

Common exploration strategies include:

\begin{itemize}

\item \textbf{$\epsilon$-Greedy:} Select a random action with probability $\epsilon$ and the greedy action with probability $1-\epsilon$.

\item \textbf{Boltzmann Exploration:} Select actions according to a softmax distribution over estimated values, with temperature controlling exploration level.

\item \textbf{Upper Confidence Bound (UCB):} Balance exploration and exploitation by preferring actions with high estimated value and high uncertainty.

\end{itemize}

\subsection{Applications and Limitations}
Reinforcement learning has achieved significant successes in game playing (Atari, Go, chess), robotics, autonomous driving, and optimization problems. Modern RL algorithms scale to high-dimensional state and action spaces and can learn complex behaviors \cite{haarnoja2018sac}.

However, RL faces several challenges:

\begin{itemize}

\item \textbf{Sample Efficiency:} RL typically requires large numbers of environment interactions, which can be expensive or infeasible in real-world settings.

\item \textbf{Reward Specification:} Defining a reward function that captures desired behavior is often difficult and can lead to unintended solutions.

\item \textbf{Generalization:} Policies learned in one environment may not transfer well to new environments or variations of the original task.

\item \textbf{Stability:} Training can be unstable, with performance varying significantly based on hyperparameters and random seeds.
\end{itemize}
